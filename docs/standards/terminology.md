# Panduan Terminologi (TS Edition)

Untuk menjaga kualitas "Gold Standard" dan nada suara (Tone of Voice) yang arsitektural, ikuti pedoman ini.

## 1. Aturan Penulisan Istilah
- **Bilingual Focus**: Gunakan istilah teknis asli dalam Bahasa Inggris (Bold), diikuti penjelasan/analogi Indonesia.
  - *Contoh*: "Sistem **Static Typing** pada TypeScript memungkinkan kita menangkap error saat kompilasi."
- **Capitalization**: Gunakan PascalCase untuk tipe data (e.g., **Interface**, **Alias**, **Generic**).

## 2. Senior vs Basic Terms
Gunakan kolom "Senior" untuk narasi profesional.

| Basic Term | Senior Terminology | Konteks |
| :--- | :--- | :--- |
| Ganti tipe | **Type Assertion** | Saat memaksa tipe data. |
| Isi data | **Property Assignment** | Saat mengisi nilai ke objek. |
| Gabung tipe | **Intersection Types** | Menggunakan operator `&`. |
| Pilihan tipe | **Union Types** | Menggunakan operator `|`. |

## 3. Metode Analogi
- **Katedral Tipe**: Fokus pada presisi konstruksi dan keamanan struktur.
- **Blueprints**: Anggap `Interface` sebagai cetak biru bangunan yang harus dipatuhi.
